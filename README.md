# ReasonLab
ReasonLab is designed with a dual purpose: it functions as a software engineering experiment to test the limits of AI-assisted development and as a cognitive science research instrument to map human thinking patterns.


Technology Stack
ReasonLab is built using a modern, high-performance stack selected for rapid prototyping and reliable data collection:

Frontend: Next.js (React) with TypeScript
Styling: Tailwind CSS for high-contrast, accessible UI
Backend & Database: Supabase (PostgreSQL)
Deployment: Vercel and GitHub


Autonomous Development Post-Mortem
This project was an experiment in AI-Led Hybrid Development (Autonomy Level 4). Using a specialized swarm of AI agents, the entire platform was planned, built, and deployed in a fraction of the time required for traditional development.

Efficiency Metrics:
Actual Human Time: 42 Hours
Estimated Manual Time: 178 Hours (based on prior coding experience)
Productivity Multiplier: 4.24x Efficiency Gain
Agent Roles: Product Manager, Cognitive Researcher, Full-Stack Engineer, Data Scientist.

Critical Lessons Learned
The Environment Problem: AI agents provided high-quality logic but initially suffered from "Context Blindness" regarding the physical folder structure (frontend/app/lib), requiring human intervention to fix relative paths.
Styling Friction: Overcoming global CSS defaults to achieve a "Pure Black" high-contrast UI required manual inline-style overrides, highlighting a current limitation in agentic UI-alignment.
The Counter-Intuitive Win: Despite manual troubleshooting with Supabase and Vercel, the multi-agent model allowed for a 400%+ speed boost, proving that AI-human partnerships are the future of scientific software engineering.
